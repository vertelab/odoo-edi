EDI AF KROM Postcode

This integration provides information that can form the basis for decisions on support. An endpoint responds on whether a postcode is covered by KROM. The API is an HTTP proxy against the underlying API of AIS. It is used to request information needed as a basis for decisions on support. More detailed flow documentation is available at the IPF API Portal: AIS BOS Regelverk.

Configuration

To configure this integration, you need to:

  1. Go to Tekniskt > EDI > EDI Route
  2. Edit or create AF Asok: Get segment route
  3. Go toProtocol settings menu and enter the following parameters
  4. Client ID and Client Secret is available in the IPF Portal

Usage

To test this integration, you need to:

  1. Go to Tekniskt > Åtgärder >Serveråtgärder
  2. Edit or create EDI AF Get KROM postcode
  3. The customer_id should be set to a jobseeker that a postcode for Västerås i.e 721 34
